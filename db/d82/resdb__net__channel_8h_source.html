35// ResDBNetChannel is used to send data to the server identified by ip:port or
36// via the provided socket. If SignatureVerifierInterface is provided, data will be
37// signed. The client will retry max_retry_time_ if the connection or sned
38// process fail. For a message with a command, it will be set inside a Request
39// message. Each Request will be put into the network message ResDBMessage and
40// signed.
41//
42// Message structre:
43// ResDBMessage:
44// Request:
45// cmd
46// user_message
47// signature
48//
49// For a raw message, it will be set inside a Network Message directly.
50// Message structre:
51// ResDBMessage:
52// user_message
53// signature
54//
55class ResDBNetChannel {
56 public:

59 // the server, it won't connect again.
